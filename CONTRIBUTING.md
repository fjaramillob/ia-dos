# Contribuir a IA-DOS

Gracias por considerar una contribución.

IA-DOS busca mantenerse simple, comprensible, reutilizable e independiente de herramientas específicas.

## Formas de contribuir

Puedes:

- reportar errores de documentación;
- proponer mejoras;
- agregar ejemplos genéricos;
- corregir enlaces;
- sugerir nuevas prácticas basadas en experiencia real;
- abrir un pull request.

## Antes de proponer un cambio importante

Abre un issue cuando la propuesta:

- agregue una regla nueva;
- cambie la estructura recomendada;
- modifique principios;
- introduzca una herramienta como recomendación oficial;
- agregue complejidad o nuevas obligaciones.

Los cambios pequeños, como errores ortográficos o enlaces rotos, pueden enviarse directamente mediante pull request.

## Criterios de aceptación

Una contribución debe:

- resolver un problema claro;
- ser aplicable a más de un proyecto;
- evitar referencias a proyectos particulares;
- no depender innecesariamente de una sola herramienta;
- explicar el motivo del cambio;
- mantener el contenido comprensible;
- evitar duplicación;
- distinguir recomendaciones de requisitos.

## Flujo recomendado

1. Crea un fork del repositorio.
2. Crea una branch para el cambio.
3. Mantén el cambio pequeño y enfocado.
4. Revisa la documentación relacionada.
5. Abre un pull request con una explicación clara.
6. Responde a las observaciones de revisión.

## Decisión final

Durante la serie `0.x`, el Lead Maintainer conserva la decisión final sobre alcance, estructura, estándares y releases oficiales.
