# Checklist de revisión fundacional

- [ ] El propósito se entiende sin conocimientos avanzados.
- [ ] El público objetivo está claramente definido.
- [ ] Los principios limitan la complejidad.
- [ ] El modelo operativo es aplicable a proyectos nuevos y existentes.
- [ ] La separación app/wiki es recomendada, no impuesta.
- [ ] Las fuentes de verdad están claras.
- [ ] Los términos técnicos están explicados.
- [ ] No existen referencias a proyectos particulares.
- [ ] No se prometen aplicaciones, agentes o automatizaciones inexistentes.
- [ ] El gobierno permite contribución sin perder control del proyecto oficial.
