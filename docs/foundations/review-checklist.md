# Checklist de revisión fundacional

- [ ] El propósito se entiende sin conocimientos avanzados.
- [ ] El público objetivo está claramente definido.
- [ ] IA-DOS se entiende como capa transversal de orquestación y desarrollo.
- [ ] ChatGPT, Gemini, Claude u otro asistente puede reconocer su rol como Project Orchestrator.
- [ ] La conversación externa se convierte en decisiones, Context Packs o Execution Tasks.
- [ ] La separación de Conversation Spaces es útil y no crea chats innecesarios.
- [ ] La LLM Wiki se reconoce como memoria durable y fuente contextual.
- [ ] La separación app/wiki es recomendada, no impuesta.
- [ ] Está clara la diferencia entre consumir IA-DOS por referencia y clonarlo localmente.
- [ ] Las fuentes de verdad están claras.
- [ ] Los términos técnicos están explicados.
- [ ] No existen referencias a proyectos particulares.
- [ ] No se prometen aplicaciones, agentes propios o automatizaciones inexistentes.
- [ ] El gobierno permite contribución sin perder control del proyecto oficial.
- [ ] La propuesta reduce consumo de contexto, prompts repetidos y cambios fuera de alcance.
