# Público objetivo

## Usuario principal

IA-DOS está pensado principalmente para personas que ya construyen productos digitales con inteligencia artificial y necesitan una forma clara de dirigir ese trabajo.

Este usuario puede no ser un programador experto, pero normalmente:

- entiende conceptos generales de sistemas;
- puede describir una necesidad funcional;
- reconoce qué es una aplicación, una API, una base de datos o un repositorio;
- utiliza ChatGPT, Gemini, Claude u otro asistente para pensar y dirigir el proyecto;
- utiliza Codex, Claude Code, Antigravity, GitHub Copilot u otro coding agent para desarrollar;
- revisa los resultados generados;
- toma decisiones de producto o tecnología;
- necesita mantener varios proyectos ordenados.

Puede describirse como un **AI-assisted builder**, es decir, una persona que construye productos digitales con apoyo de inteligencia artificial.

## Interfaz principal

Para este usuario, IA-DOS no comienza necesariamente dentro del editor de código.

Puede comenzar en:

- un Project de ChatGPT;
- un Gem de Gemini;
- un Project de Claude;
- una conversación persistente;
- un asistente personalizado.

Ese espacio funciona como orquestador del proyecto. Ayuda a:

- comprender dónde está parado el proyecto;
- separar conversaciones clave;
- mantener dirección y prioridades;
- registrar decisiones;
- preparar inputs claros para desarrollo;
- reducir contexto repetido y consumo innecesario de tokens;
- interpretar los reportes de los coding agents;
- decidir el siguiente paso.

## Perfiles que pueden beneficiarse

IA-DOS puede ser útil para:

- fundadores y emprendedores que desarrollan productos digitales;
- product builders y perfiles híbridos entre negocio y tecnología;
- analistas y consultores que crean automatizaciones o integraciones;
- desarrolladores individuales que utilizan varios agentes;
- equipos pequeños sin procesos formales de ingeniería;
- personas que administran varios proyectos al mismo tiempo.

## Conocimientos mínimos esperados

No se exige experiencia avanzada en programación. Sí es recomendable comprender, al menos de forma general:

- qué es un repositorio;
- qué diferencia existe entre desarrollo local y producción;
- qué es una tarea de desarrollo;
- por qué un cambio debe probarse;
- por qué deben protegerse secretos y credenciales;
- que un agente puede equivocarse aunque su respuesta parezca convincente;
- que una conversación no debe ser la única memoria del proyecto.

IA-DOS debe explicar los conceptos técnicos necesarios sin asumir formación senior.

## Para quién no está diseñado principalmente

IA-DOS no es:

- un curso de programación desde cero;
- un evaluador de ideas de negocio;
- una metodología exclusiva para grandes equipos;
- una plataforma de agentes autónomos;
- un reemplazo de especialistas en seguridad, arquitectura o infraestructura cuando el riesgo exige su participación.

## Promesa al usuario

IA-DOS no promete construir cualquier aplicación automáticamente.

Su promesa es más concreta:

> Ayudar a convertir conversaciones e intención en desarrollo estructurado, manteniendo el control del contexto, las decisiones, los agentes y la evidencia.
