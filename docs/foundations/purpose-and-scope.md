# Propósito y alcance

## Propósito

IA-DOS es una capa operativa común para proyectos de software desarrollados con asistencia de inteligencia artificial.

Su función es ayudar a ordenar:

- el workspace local;
- la aplicación;
- la memoria del proyecto;
- las instrucciones para agentes;
- las tareas;
- las decisiones;
- la verificación;
- el registro del trabajo realizado.

IA-DOS busca que una persona pueda utilizar distintos agentes sin depender de conversaciones aisladas ni perder el control del proyecto.

## Problema que aborda

Los agentes pueden producir código con rapidez, pero no conocen por sí solos:

- el propósito completo del proyecto;
- qué decisiones siguen vigentes;
- qué zonas pueden modificar;
- qué riesgos deben evitar;
- qué información es real y cuál está planificada;
- qué evidencia se necesita para considerar una tarea terminada.

Sin una estructura común, el desarrollo puede acumular bugs, código innecesario, documentación contradictoria y consumo excesivo de contexto.

## Alcance de IA-DOS

IA-DOS define una forma de trabajo que puede aplicarse a:

- aplicaciones web y móviles;
- APIs e integraciones;
- automatizaciones;
- agentes y chatbots;
- herramientas internas;
- plataformas SaaS;
- sistemas de datos;
- proyectos nuevos;
- proyectos existentes.

El framework recomienda prácticas y estructuras. No impone un lenguaje, framework, proveedor cloud o agente específico.

## Fuera de alcance

IA-DOS no:

- evalúa si una idea debe construirse;
- define el modelo de negocio;
- reemplaza la dirección de producto;
- reemplaza la revisión humana;
- programa de forma autónoma;
- garantiza ausencia de errores;
- sustituye Git, GitHub, Obsidian o las herramientas de desarrollo;
- obliga a reorganizar un proyecto existente sin evaluar el impacto.

## Alcance de `v0.1.0-alpha.1`

La primera versión define:

- propósito;
- público objetivo;
- principios;
- modelo operativo básico;
- terminología;
- estructura recomendada del workspace;
- forma inicial de adopción;
- gobierno y contribución.

La instalación práctica, los prompts y las plantillas se agregarán en incrementos separados dentro de la misma versión alpha.
