# No objetivos

IA-DOS no busca:

- reemplazar herramientas de desarrollo;
- imponer un stack tecnológico;
- automatizar todo el ciclo de desarrollo;
- evaluar la viabilidad comercial de proyectos;
- sustituir especialistas cuando el riesgo exige experiencia profesional;
- convertir cada proyecto en una estructura idéntica;
- crear procesos por cumplir una formalidad;
- centralizar secretos o credenciales;
- depender de una sola empresa o agente.

Estos límites protegen el alcance del framework.
