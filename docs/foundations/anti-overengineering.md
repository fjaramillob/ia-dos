# Evitar sobreingeniería

IA-DOS no debe convertirse en una nueva fuente de burocracia.

## Señales de alerta

- crear documentos que nadie consulta;
- duplicar la misma información;
- exigir procesos avanzados a proyectos pequeños;
- agregar herramientas antes de validar una necesidad;
- convertir recomendaciones en obligaciones sin justificación;
- diseñar automatizaciones antes de estabilizar el proceso manual.

## Regla práctica

Una nueva práctica debe responder:

1. ¿Qué problema concreto resuelve?
2. ¿En qué proyectos fue útil?
3. ¿Qué coste agrega?
4. ¿Puede aplicarse de forma más simple?
5. ¿Debe ser obligatoria, recomendada u opcional?

Si estas preguntas no tienen una respuesta clara, la práctica todavía no pertenece al estándar común.
